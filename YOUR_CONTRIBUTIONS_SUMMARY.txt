SAMPLE RESPONSES FOR TECHNICAL QUESTIONS:
===============================================

Q: "How does shared alarm synchronization work?"
A: "Shared alarms use a hub-and-spoke model where one user creates the alarm 
and invites others. We store a main alarm time in Firestore and each user 
can have individual time offsets. Firebase Cloud Functions handle the 
synchronization and notification delivery to ensure all participants 
receive notifications simultaneously, accounting for timezone differences."

Q: "Why did you choose this architecture?"
A: "I focused on reliability and user experience. Firebase provides real-time 
synchronization and offline support. The controller-based architecture 
separates business logic from UI, making it easier to test and maintain. 
Cloud Functions ensure backend operations continue even if the app is closed."

Q: "How do you handle edge cases?"
A: "I implemented comprehensive error handling - user authentication validation, 
network connectivity checks, timezone conversion verification, and fallback 
mechanisms. The debug screen provides detailed logging for troubleshooting, 
and the notification system includes retry logic for failed deliveries."
