1. 🔄 NEGATIVE CONDITION ALARMS
================================================================

WHAT IT DOES:
Allows alarms to ring OR cancel based on inverted conditions (e.g., "cancel when 
weather is sunny" instead of "ring when weather is sunny").

HOW YOU IMPLEMENTED IT:
- Activity Conditions: 4 types - Ring/Cancel when Active/Inactive within time interval
- Weather Conditions: Ring when weather matches vs. cancel when weather matches  
- Location Conditions: Ring when at location vs. cancel when at location

KEY IMPLEMENTATION (AlarmReceiver.kt):
```kotlin
when (activityConditionType) {
    1 -> { // Ring when active
        shouldRing = isActiveWithinInterval
    }
    2 -> { // Cancel when active (negative condition)
        shouldRing = !isActiveWithinInterval
    }
    3 -> { // Ring when inactive
        shouldRing = !isActiveWithinInterval
    }
    4 -> { // Cancel when inactive (negative condition)
        shouldRing = isActiveWithinInterval
    }
}
```

CHALLENGES SOLVED:
- Complex Logic Management: Created comprehensive condition evaluation system
- User Understanding: Made inverted logic intuitive with clear UI descriptions
- Edge Case Handling: Proper handling of "all types selected" scenarios

```dart
Future<void> sendDirectNotificationToSharedUsers(AlarmModel alarmData) async {
  // Method 1: Cloud Functions notification
  final sharedItem = {
    'type': 'alarm',
    'AlarmName': alarmData.firestoreId,
    'owner': alarmData.ownerName ?? 'Someone',
    'alarmTime': alarmData.alarmTime
  };
  await PushNotifications().triggerSharedItemNotification(
    alarmData.sharedUserIds!, sharedItem: sharedItem
  );
  
  // Method 2: Direct Firestore notifications
  for (String userId in alarmData.sharedUserIds!) {
    await FirebaseFirestore.instance
      .collection('userNotifications')
      .doc(userId)
      .collection('notifications')
      .add({
        'type': 'alarm_update',
        'title': 'Shared Alarm Updated! 🔔',
        'message': '${alarmData.ownerName} updated the alarm time to ${alarmData.alarmTime}',
      });
  }
}
```

4. 🌍 TIMEZONE-AWARE ALARMS
================================================================

WHAT IT DOES:
Allows users to set alarms in different timezones that automatically adjust for 
DST and convert to local time for proper scheduling.

HOW YOU IMPLEMENTED IT:
- Bidirectional Conversion: User timezone ↔ Device timezone conversion
- DST Handling: Automatic daylight saving time adjustments
- Smart Scheduling: Converts target timezone to local time for Android AlarmManager

KEY IMPLEMENTATION (add_or_update_alarm_controller.dart):
```dart
DateTime getTimezoneAwareAlarmTime() {
  if (!isTimezoneEnabled.value || selectedTimezoneId.value.isEmpty) {
    return selectedTime.value;
  }

  try {
    final targetLocation = tz.getLocation(selectedTimezoneId.value);
    final localLocation = tz.getLocation(deviceTimezoneId.value);
    
    // Create proper TZDateTime in target timezone
    final targetTZDateTime = tz.TZDateTime(
      targetLocation,
      selectedTime.value.year, selectedTime.value.month, selectedTime.value.day,
      selectedTime.value.hour, selectedTime.value.minute,
    );
    
    // Convert to local timezone for scheduling  
    final localTZDateTime = tz.TZDateTime.from(targetTZDateTime, localLocation);
    
    return DateTime(localTZDateTime.year, localTZDateTime.month, localTZDateTime.day,
                   localTZDateTime.hour, localTZDateTime.minute);
  } catch (e) {
    return selectedTime.value; // Fallback
  }
}
```

CHALLENGES SOLVED:
- DST Complexity: Automatic handling of daylight saving transitions
- UI Confusion: Clear display of timezone conversion in real-time
- Scheduling Accuracy: Precise local time calculation for alarm triggers

5. 📅 GOOGLE CALENDAR INTEGRATION
================================================================

WHAT IT DOES:
Syncs with Google Calendar to import events and create alarms automatically 
from calendar reminders.

HOW YOU IMPLEMENTED IT:
- OAuth Integration: Google Sign-In with Calendar API scopes
- Event Import: Calendar list retrieval and event parsing
- Alarm Creation: Automatic alarm generation from calendar events

KEY IMPLEMENTATION (google_cloud_api_provider.dart):
```dart
static Future<List<CalendarListEntry>?> getCalenders() async {
  if (_googleSignIn.currentUser == null) {
    await getInstance();
  }
  final authHeaders = await _googleSignIn.currentUser!.authHeaders;
  final httpClient = GoogleHttpClient(authHeaders);
  var dataList = await CalendarApi(httpClient).calendarList.list();

  return dataList.items;
}

static Future<List<Event>?> getEvents(String calenderId) async {
  await getInstance();
  final authHeaders = await _googleSignIn.currentUser!.authHeaders;
  final httpClient = GoogleHttpClient(authHeaders);
  var dataList = await CalendarApi(httpClient).events.list(calenderId);
  return dataList.items;
}
```

CHALLENGES SOLVED:
- Authentication Flow: Secure OAuth with proper scope management
- Data Synchronization: Real-time calendar event updates
- User Experience: Seamless integration without disrupting existing workflows
